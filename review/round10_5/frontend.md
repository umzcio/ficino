## Regressions/Surprises

- **No regressions found.** All 21 findings resolved; no evidence any fix reintroduced a Round-9 or
  Round-10 defect class in the surrounding code touched by the same commit.
- **FE-9 surprise:** the fix is not fully clean — `ParentPostCard`'s `aria-label` is identical for the
  actual-parent-post use and the quoted-post use (both say "View parent post by {name}"), even though
  the surrounding UI visually distinguishes them (an "Original post" heading precedes the quoted-post
  instance, PostDetail.tsx:178). This was flagged as a minor at wave-4 final review
  ("FE-9 label prop parent-vs-quoted differentiation") and never followed up — worth a one-line R10.5
  fix (thread an `aria-label` prop through `ParentPostCard` differentiating "parent" vs "quoted").
- **FE-1/FE-10 surprise:** both were fixed a full wave before the "R10 Wave 4 — Frontend" plan (which
  explicitly scopes itself to "FE-2..9, FE-11..21") even started — FE-1 landed in Wave 1 commit `2c9a3f1`
  (bundled with the vitest-infra task) and FE-10 in Wave 1 commit `f275a6a` (filed as DEP-2). Neither
  regressed since; both still hold at HEAD.
- **FE-4/FE-21 surprise (positive):** both fixes exceed their finding's literal ask — FE-4's group-chat
  flow gained a 404-retry/backoff loop after a mid-wave review caught a 100%-reproducible failure mode
  the finding didn't originally describe (freshly created groups had no server-side placeholder row);
  FE-5/FE-4's siblings both added an IDB offline-fallback read the finding only "optionally" recommended.
